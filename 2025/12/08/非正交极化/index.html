Dipole Moment of Non-orthogonal Lattice

Created2025-12-08|Updated2025-12-09|Diary
|Post Views:

It is known that dipole moment uing Berry Phase method given by VASP is under Cartesian coordinate system. While in non-orthogonal lattice, handling the dipole moment multivalue is difficult only by dipole moment in Cartesian coordinate system.

Considering polarization P in a non-orthogonal lattice with lattice vectors a, b, c, the dipole moment components in Cartesian coordinate system are given by VASP as P = (Px, Py, Pz). Suppose dipole moment components in basis vector coordinate system are (Pa, Pb, Pc), then the relation between dipole moment in Cartesian coordinate system and basis vector coordinate system is given by

(a,b,c)(PaPbPc)=(PxPyPz)⇒(PaPbPc)=(a,b,c)−1(PxPyPz)\left ( \mathbf{a} ,\mathbf{b}, \mathbf{c}\right ) \begin{pmatrix}P_a\\P_b\\P_c\end{pmatrix} =\begin{pmatrix}P_x\\P_y\\P_z\end{pmatrix} \Rightarrow \begin{pmatrix}P_a\\P_b\\P_c\end{pmatrix} =\left ( \mathbf{a} ,\mathbf{b}, \mathbf{c}\right ) ^{-1}\begin{pmatrix}P_x\\P_y\\P_z\end{pmatrix} (a,b,c)​Pa​Pb​Pc​​​=​Px​Py​Pz​​​⇒​Pa​Pb​Pc​​​=(a,b,c)−1​Px​Py​Pz​​​

In this way, we can get dipole moment in basis vector coordinate system, where the multivalue problem can be easily handled by adding integer multiples of eR/Ω along each lattice vector direction. In our case, we can add or subtract integer values from each component of (Pa, Pb, Pc) to obtain the correct dipole moment change branch during a process.

For example, for In2Se3, belongs to hexagonal lattice system. https://www.nature.com/articles/ncomms14956

a=(4.1114,0,0)b=(−2.0557,3.56058,0)c=(0,0,29.1318)\mathbf{a}=(4.1114, 0, 0)\quad \mathbf{b}=(-2.0557, 3.56058, 0)\quad \mathbf{c}=(0, 0, 29.1318) a=(4.1114,0,0)b=(−2.0557,3.56058,0)c=(0,0,29.1318)

The first three clumns are the original data that have not undergone multivalue processing. Let’s do a thought experiment: for such a hexagonal lattice, if we move one electron from one unit cell to the neighboring unit cell along x direction in Cartesian coordinate system, when Δx=4.1114, the system returns to the same state. similarly, along y direction, when Δy=3.56058, the system returns to the same state. Therefore, we can arbitrarily add or subtract integer multiples of 4.1114 along x direction and 3.56058 along y direction to the dipole moment data in Cartesian coordinate system to obtain the correct dipole moment change branch. The last three columns are the dipole moment data after multivalue processing.

NOTE:
1.The insertation from is to fs should be continuous;
2.Use vasp6.3 version or above to run the calculation, otherwise the dipole moment data in OUTCAR file will be wrong.(vasp5.4.4 version has been tested to give wrong ionic dipole moment).
